- `IO-005` 接收格式切换：ASCII/HEX/BIN/DEC 同源渲染（计划：2026-04-15，状态：已完成）
- `IO-006` 时间戳：接收行支持 `[时:分:秒.毫秒]` 绝对时间；可通过「按时间显示」开关关闭时间戳（与 `IO-009` 联动）（计划：2026-04-15，状态：已完成）
- `IO-007` 统计：发送/接收字节、帧数、错误数、清零（计划：2026-04-16，状态：已完成）
- `IO-008` 一键清空：接收区/发送区/统计独立清空（计划：2026-04-16，状态：已完成）
- `IO-009` 接收展示双开关（左栏 RTS 下方，按压样式同串口开关）：
  - **按时间显示**：开 = 每行前加 `[时:分:秒.毫秒]`；关 = 仅 payload，按块顺序追加。
  - **自动换行**：开 = 两次接收间隔大于阈值（默认 350ms，大于轮询间隔）时在下一段前插入换行，且每块行尾换行；关 = 不在块之间插入换行，无对端 `0D`/`0A` 时持续拼接为流；暂停恢复时与开关一致（`\n` 拼接 vs 无分隔拼接）。
  - **四种组合状态**（2×2）：

| 按时间 | 自动换行 | 接收区表现（摘要） |
|--------|----------|----------------------|
| 开 | 开 | 时间戳 + 分行；停顿大时额外分段换行 |
| 开 | 关 | 时间戳 + 块间不插换行（仍可能因对端字节内含换行而换行） |
| 关 | 开 | 无时间戳 + 分行与停顿分段 |
| 关 | 关 | 无时间戳 + 纯流式拼接 |

  （状态：已完成）

- `IO-010` 接收区滚动策略：默认跟随最新；用户上滑离开底部则暂停自动滚动；滚回底部恢复跟随；仅在接收区右侧滚动条区域点击时可出现浮动「回到最新」（状态：已完成）
- `IO-011` 全局滚动：页面外层不出现纵向滚动条，仅接收区等需要的区域滚动（状态：已完成）
- `IO-012` 发送换行下拉文案：选项展示实际发送字节含义（如 `\\r\\n (0D 0A)`、`\\n (0A)`、`\\r (0D)`），不单独写抽象 `CRLF`（状态：已完成）
- `IO-013` ASCII 接收可见化：控制字符与解码异常时不显示为「空白」，必要时回退为可见字节表示（状态：已完成）

## 推荐统一类型（前后端共享）

```ts
export type PortInfo = {
  id: string
  path: string
  friendlyName: string
  status: "idle" | "occupied" | "opened" | "disconnected" | "error"
}

export type SerialConfig = {
  baudRate: number
  dataBits: 5 | 6 | 7 | 8
  stopBits: 1 | 1.5 | 2
  parity: "none" | "odd" | "even" | "mark" | "space"
  flowControl: "none" | "rtscts" | "xonxoff"
  dtr: boolean
  rts: boolean
}
```
